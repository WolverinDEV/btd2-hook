# Player profile data
The players' data is stored under the filename `Profile.save`.  
The file contains a JSON encoded object containing the following keys:
- `currencies` An object containg the following keys:
  - `monkey_money` A number representing the current monkey money.
  Note: When incrementing this value only "possible" values are allowed (e.g. the amount you can buy ingame)
  - `battle_points` A number representing the current amount of battle points.
  - `universal_xp` A number containing the amount of universal XP
- `towers` An array of [tower objects](...)
- `tower_animations` An array of [tower animation objects](...)
- `tower_loadout` A size 4 tuple of strings with tower names representing the last player loadout
- `avatars` An array of [avatar objects](...)
- `avatar_loadout` An object containing the following keys
  - `equipped_avatar` A string containing the currently equipped avatar
  - `equipped_card_banner` A string containing the currently equipped card banner
  - `equipped_card_border` A string containing the currently equipped card border
- `emotes` An array containing [emote objects](...)
- `emote_loadout` An object containing the following keys, representing the current emoji loadout
  - `equipped_visual_emote_0` .. `equipped_visual_emote_9` Visual emotes
  - `equipped_speech_emote_0` .. `equipped_speech_emote_3` Speech emotes
- `initial_profile_game_version` A number containing the initial game version when the profile has been created
- `version` A number containing the current profile version (Current is 4)
- `last_save_game_version` A number containing the last game version with which the profile has been saved
- `birthday_ts` A string containing the timestamp of the players birthday (entered when creating the account)
- `accepted_tos_ts` A string containing the timestamp when the player accepted the TOS
- `ladder` An [ladder object](...)
